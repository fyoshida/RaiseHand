・方針
  ・手をあげる、おろす、サポートはStudentで行う
  ・IP、HostNameはチェックしてから使用
  ・Session
    ・最終Load日時
    ・StudentManager
    ・OrderManager
  ・ServletContext（最終更新日時）
  ・リポジトリ
    ・更新時
      ・順位更新後、Sessionに保存し、Repositoryにも保存
    ・参照時
      ・SessionになければリポジトリからLoad
      ・最終Load時＜最終更新日ならリポジトリからLoad
      ・最終Load時＞最終更新日ならSessionからLoad
    ・Load、Save時に渡す物
      ・StudentList
      ・OrderList
    ・保存テーブル
      ・PcList(IP,HostName,isStudent)
      ・StudentList(IP,HostName,UserName,HelpStatus,handUpTime)
      ・OrderList(Order、IP)
    ・種類
      ・DataBase
      ・File
      ・InMemory（ServletContext）
  ・読み込み後
    ・StudentManager：学生検索用
    ・OrderManager：順位更新用
      ・ListとMapを使用（MapはLoad時に設定）
  ・NetworkInterface
    ・Network版、Dummy版
    
  ・テスト
    ・Unit：自分以外は全てMock
    ・統合テスト：複数クラス
    ・ServiceTest：Servlet
    ・UIテスト：Selenium
